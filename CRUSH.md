# crush
This file contains information about how to build, test, and contribute to this repository.

## Commands
- Run the app: `flutter run`
- Run tests: `flutter test`
- Run a single test file: `flutter test test/widget_test.dart`
- Build the app: `flutter build`
- Analyze code: `flutter analyze`

## Code Style
- Follow the official [Dart style guide](https://dart.dev/guides/language/effective-dart/style).
- Use `PascalCase` for class names.
- Use `camelCase` for variable and function names.
- Use single quotes for strings.
- Add types to all variables and function signatures.
- Use `final` for variables that are not reassigned.
- Use `const` for compile-time constants.
- Imports should be sorted alphabetically and grouped by package.
- Handle errors gracefully using `try-catch` blocks.
- Use `async/await` for asynchronous operations.
- All new features should have corresponding tests.
- All new widgets should be in their own file.
- All new screens should be in the `screens` directory.
- All new controllers should be in the `controllers` directory.
- All new models should be in the `models` directory.
